## A. 선행 확인 필수 사항 (구축 전 반드시 확인)

아래 2건은 **확인되지 않으면 수도권 데이터 적재가 불가능**합니다.

### A-1. 수도권 수단 TXT 컬럼 매핑 검증

수도권 수단별 OD 텍스트 파일(`OD_MMODE_23_F.TXT` 등)은 **열 이름(헤더)이 없습니다.** 비수도권 Excel 컬럼과 값 크기를 비교하여 아래와 같이 추정했으나, 정확한 매핑은 설명자료 확인이 필요합니다.

| 열 위치 | 추정 수단 | 추정 근거 | 샘플값 | 확인 결과 |
|---------|----------|----------|--------|----------|
| col5 | 도보/자전거 | 비수도권 대응, 값이 큼 | 5242.727 | ☐ 맞음 / ☐ 틀림 → ______ |
| col6 | 승용차 | 비수도권 대응 | 201.206 | ☐ 맞음 / ☐ 틀림 → ______ |
| col7 | 버스 | 비수도권 대응 | 325.747 | ☐ 맞음 / ☐ 틀림 → ______ |
| col8 | 일반철도 | 대부분 0, 비수도권 대응 | 0.0 | ☐ 맞음 / ☐ 틀림 → ______ |
| col9 | 고속철도 | 대부분 0, 비수도권 대응 | 0.0 | ☐ 맞음 / ☐ 틀림 → ______ |
| col10 | 도시철도(지하철) | 비수도권 대응 | 729.808 | ☐ 맞음 / ☐ 틀림 → ______ |

### A-2. 수도권 "업무" 목적 파일 누락 여부

수도권 목적별 주수단 OD(MOD-21 ZIP) 안에 4개 목적 파일만 있고 **"업무" 파일이 없습니다.**

| 목적 | 파일명 패턴 | 존재 여부 |
|------|------------|----------|
| 출근 | `OD_MMODE_WORK_{YY}_F.TXT` | ✅ 있음 |
| 등교 | `OD_MMODE_SCHO_{YY}_F.TXT` | ✅ 있음 |
| 귀가 | `OD_MMODE_HOME_{YY}_F.TXT` | ✅ 있음 |
| 기타 | `OD_MMODE_OTHE_{YY}_F.TXT` | ✅ 있음 |
| **업무** | `OD_MMODE_BUSI_{YY}_F.TXT` ? | **❌ 없음** |

**질문**: 업무 목적 파일이 없는 것이 KTDB의 의도적 미제공인가요, 아니면 누락인가요?

> 회신: ☐ 의도적 미제공 (업무 수단별 분석 불필요) / ☐ 누락 (KTDB에 요청 필요) / ☐ 확인 필요

## B. 설계 판단 필요 사항 (구축 방향 결정)

아래 8건은 **어느 쪽을 선택해도 구축은 가능**하지만, 분석 활용 방식에 영향을 주므로 설계엔지니어의 판단이 필요합니다.

### B-1. 수단 분류 통합 여부

전국/비수도권/수도권에서 **같은 수단이 다른 이름**으로 되어 있습니다.

| 수단 | 전국 250존 | 비수도권 | 비고 |
|------|-----------|---------|------|
| 도시철도 | `지하철` | `도시철도` | 이름만 다름 |
| 철도 | `일반철도` + `고속철도` (분리) | `일반/고속철도` (합산) | 분리 vs 합산 |
| 도보+자전거 | 없음 | `도보/자전거` | 전국에는 해당 수단 자체가 없음 |
| 택시 | 없음 | `택시` | 전국에는 해당 수단 자체가 없음 |

**질문**: 전국과 권역을 하나로 합쳐서 비교 가능하게 할까요, 원본 그대로 유지할까요?

> 회신: ☐ (a) 통합 표준화 (전국-권역 비교 분석 가능) / ☐ (b) 원본 유지 (정확성 우선)

### B-2. 파생 컬럼(합산값) 포함 여부

비수도권 OD Excel에는 원본 수단 외에 **합산 컬럼**이 포함되어 있습니다.

- `승용차/택시` = 승용차 + 택시의 합
- `기타합계` = 기타 수단의 합

이 값들은 원본 컬럼에서 계산 가능하므로 중복 데이터입니다.

> 회신: ☐ (a) 제외 (중복이므로 저장 불필요) / ☐ (b) 포함 (보고서에 해당 합산값을 직접 사용)

### B-3. 17존 피벗 시트 활용

전국 250존 OD Excel에는 시도(17존) 간 OD를 행렬(매트릭스)로 요약한 **"대존" 시트**가 있습니다.

> 회신: ☐ (a) 별도 요약 테이블로 저장 (시도간 요약표를 자주 사용) / ☐ (b) 무시 (필요 시 250존에서 집계)

### B-4. 목적별 주수단 OD 포함 범위

수도권은 목적×수단 OD가 7개년(2023~2050) 제공되지만, 비수도권 접근수단 OD는 **2023년 단일 연도**만 제공됩니다.

> 회신: ☐ (a) 전부 포함 (접근수단 분석도 필요) / ☐ (b) 1차 제외 (우선순위 낮음)

### B-5. 접근수단 OD 포함 여부

"접근수단 OD"는 일반 수단별 OD와 **개념이 다릅니다** — 철도역까지 어떤 수단으로 갔는지를 나타냅니다. (예: 철도를 타기 위해 버스로 역까지 이동한 통행량)

> 회신: ☐ (a) 포함 (철도 접근성 분석 필요) / ☐ (b) 제외 (1차 범위에서 불필요)

### B-6. Excel 수식 셀 처리 방식

일부 Excel 파일의 "합계" 컬럼이 숫자가 아닌 **수식(`=SUM(...)`)** 으로 저장되어 있습니다.

> 회신: ☐ (a) 캐시값 읽기 (빠르지만 Excel에서 한번 열어야 캐시 생성) / ☐ (b) 직접 계산 (안전하지만 처리 시간 증가)

### B-7. 과거년도 데이터 포함 여부

2023년 기준으로 배포된 수도권 OD 3건(구버전)이 별도로 존재합니다. 최신은 2024년 기준 배포입니다.

> 회신: ☐ (a) 포함 (2023 vs 2024 기준 비교 분석 필요) / ☐ (b) 최신(2024 기준)만 사용

### B-8. 분석 우선순위 (Google AI Studio 활용 시)

Google AI Studio에 데이터를 올릴 때, 전체 9,300만 행을 한번에 올릴 수 없어 **우선순위대로 단계적 업로드**가 필요합니다.

| 순서 | 데이터 | 행 수 | 활용도 |
|------|--------|-------|--------|
| 1순위 | 전국 250존 목적별 OD | ~44만 | 시군구간 통행량, 수단분담률 |
| 1순위 | 전국 250존 수단별 OD | ~44만 | 수단별 추이 분석 |
| 2순위 | 화물 OD | ~44만 | 물류 분석 |
| 3순위 | 비수도권 권역별 OD | ~1,850만 | 권역 내 읍면동 상세 분석 |
| 4순위 | 수도권 OD | ~7,200만 | 수도권 상세 분석 (변환 어려움) |
