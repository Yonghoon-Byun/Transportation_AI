# KTDB 교통 데이터 AI 분석 — 구축 가능성 평가 및 직접 구축 가이드

**작성일**: 2026-03-23 | **대상**: 교통 설계엔지니어 | **작성**: AX팀

---

## [1] 개발 가능성 평가

### 무엇을 만드는가

KTDB에서 제공하는 **ZIP 25건(약 9,300만 행)**의 교통 OD 데이터를 AI가 이해할 수 있는 형태로 정리하고, **자연어로 질의**하면 즉시 분석 결과를 반환하는 시스템입니다.

```
현재:  ZIP 다운로드 → Excel 수작업 → 데이터 찾기 수십분 → 보고서 작성 8시간+
목표:  데이터 정리 → Google AI Studio에 업로드 → 자연어 질문 → 즉시 표+해석 제공
```

### 가능성 판단: ✅ 구축 가능 (설계엔지니어가 직접 수행 가능)

| 평가 항목 | 판정 | 근거 |
|----------|------|------|
| 원본 데이터 분석 | ✅ 완료 | ZIP 25건 파일 포맷·컬럼 전수 분석 완료 |
| 데이터 정제 방법 | ✅ 확립 | Excel → CSV 변환 절차 정의됨 |
| AI 도구 선정 | ✅ 확정 | Google AI Studio (Gemini) — 무료, 파일 업로드 지원 |
| 수도권 TXT 수단 매핑 | ⚠️ 부분 | 10개 중 7개 추정 완료, 3개 미확인 (설명자료 검증 필요) |

### 왜 Google AI Studio + Gemini인가

| 비교 항목 | 전통적 DB 구축 | Google AI Studio (Gemini) |
|----------|--------------|--------------------------|
| 필요 인력 | 개발자 + DBA 필수 | **설계엔지니어 단독 가능** |
| 초기 세팅 | DB 서버 설치, DDL 작성, ETL 개발 | **Google 계정만 있으면 시작** |
| 데이터 업로드 | SQL COPY 명령어, 파서 코딩 | **CSV 파일 드래그 앤 드롭** |
| 질의 방식 | SQL 작성 필요 | **자연어로 질문** |
| 비용 | 서버 운영비 월 수만~수십만원 | **무료** (일일 사용량 제한 있음) |
| 데이터 용량 제한 | 제한 없음 | **파일당 최대 용량 제한 있음** → 분할 업로드 |

### 알아야 할 핵심 사항 4가지

| # | 사항 | 내용 | 왜 중요한가 |
|---|------|------|------------|
| 1 | **합계 행 제거** | 원본에 "합계" 행이 포함 → AI가 SUM 시 2배 오류 발생 | CSV 변환 시 합계 행을 삭제하거나 별도 표시 필요 |
| 2 | **수단명 불일치** | "지하철"(전국) = "도시철도"(권역) — 같은 수단인데 이름이 다름 | AI에게 "지하철과 도시철도는 같은 수단" 이라고 알려줘야 함 |
| 3 | **데이터 단위** | 여객 OD = 통행/일(AAWDT), 화물 OD = **대/일** | 혼용하면 보고서 수치 오류 |
| 4 | **용량 분할** | 수도권 OD 4,800만 행은 한번에 업로드 불가 | 연도별 또는 권역별로 CSV를 나눠서 업로드 |

### 기대 효과

| 항목 | 현재 | 목표 |
|------|------|------|
| 데이터 준비 | ~8시간 | ~1시간 (CSV 변환 + 업로드) |
| 분석 질의 | 수십분 (Excel 수작업) | **수초 (자연어 질문)** |
| 보고서 작성 | 기준 | **2배 향상** (표·해석 자동) |

## [2] 직접 구축하는 방법 — Google AI Studio 활용

### 전체 흐름 (3단계)

```
Step 1. 데이터 준비        Step 2. AI Studio 설정       Step 3. 자연어 분석
(Excel → CSV 변환)        (업로드 + 지시문 작성)       (질문 → 즉시 결과)
      ↓                          ↓                          ↓
  1~2시간 소요              30분 소요                   즉시 사용 가능
```

### Step 1. 데이터 준비 (Excel → CSV 변환)

**대상 파일 우선순위** — 전체 25건을 한번에 할 필요 없습니다. 아래 순서로 필요한 것부터 시작하세요.

| 순서 | 데이터 | 원본 파일 | CSV 변환 후 크기 | 난이도 |
|------|--------|----------|-----------------|--------|
| 1순위 | **전국 250존 목적별 OD** | `전국_목적별OD(250).xlsx` | 약 44만 행 (~30MB) | 쉬움 |
| 1순위 | **전국 250존 수단별 OD** | `전국_주수단별OD(250).xlsx` | 약 44만 행 (~30MB) | 쉬움 |
| 2순위 | **화물 OD** | `화물자동차OD.xlsx` | 약 44만 행 (~25MB) | 쉬움 |
| 3순위 | **비수도권 권역별 OD** | `{권역명}_목적통행.xlsx` | 권역당 수십만 행 | 보통 |
| 4순위 | 수도권 OD | `.OUT`, `.TXT` 파일 | 수백만~수천만 행 | **어려움** (별도 도구 필요) |

**CSV 변환 방법 (1순위 기준)**:

```
1. ZIP 압축 해제
2. Excel 파일 열기 (예: 전국_목적별OD(250).xlsx)
3. 연도별 시트(2023_목적OD, 2025_목적OD, ...) 각각에 대해:
   a. "합계" 행 삭제 (⚠️ 중요! — 목적이 "합계"인 행 전부 삭제)
   b. 빈 컬럼(None) 삭제
   c. 1행에 컬럼명 확인: 출발시도, 도착시도, 출발시군구, 도착시군구, 출근, 등교, 업무, 귀가, 기타
   d. "연도" 컬럼 추가하여 해당 연도값 입력 (예: 2023)
   e. 다른 이름으로 저장 → CSV(쉼표로 분리)(*.csv) 선택
4. 7개 연도 CSV를 하나로 합치기 (Excel에서 복사-붙여넣기)
```

> **수단별 OD 변환 시 추가 작업**: "합계" 행 외에, 전국 250존 수단 OD에서는 `합계` 수단(mode='합계')도 삭제해야 합니다.

### Step 2. Google AI Studio 설정

**접속**: [aistudio.google.com](https://aistudio.google.com) → Google 계정 로그인

```
1. 좌측 메뉴 → "새 프롬프트 만들기" 클릭
2. 모델 선택: Gemini 2.5 Pro (최신 모델 권장)
3. "System Instructions" (시스템 지시문)에 아래 내용 붙여넣기
4. 파일 업로드: 📎 아이콘 → Step 1에서 만든 CSV 파일 업로드
```

**시스템 지시문 (복사하여 붙여넣기)**:

```
너는 KTDB(국가교통 데이터베이스) 교통 분석 전문가야.
업로드된 CSV 파일은 전국 250존(시군구) 기준 OD(Origin-Destination) 통행량 데이터야.

[데이터 구조]
- 출발시도/도착시도: 17존(시도) 코드 (1=서울, 2=부산, ... 17=제주)
- 출발시군구/도착시군구: 250존(시군구) 코드 (1~250)
- 통행량 단위: AAWDT (연평균 평일 일통행량, 통행/일)
- 연도: 2023(현황), 2025~2050(장래 추정)

[목적별 OD 컬럼]
출근, 등교, 업무, 귀가, 기타 (합계 행은 이미 제거됨)

[수단별 OD 컬럼]
승용차, 버스, 지하철, 일반철도, 고속철도, 항공, 해운

[주의사항]
- 합계 행은 제거된 상태이므로 SUM 집계 시 이중 계산 걱정 없음
- "지하철"과 "도시철도"는 같은 수단 (권역 데이터에서는 "도시철도"로 표기)
- 여객 OD 단위는 통행/일, 화물 OD 단위는 대/일 — 절대 혼용하지 마
- AAWDT는 평일 기준이므로 연평균(AADT)과 다름

[응답 형식]
- 항상 표 형태로 결과를 먼저 보여주고, 아래에 해석을 붙여줘
- 수치는 천 단위 쉼표 표기 (예: 12,345)
- 단위를 반드시 명시해 (통행/일, 대/일)
```

### Step 3. 자연어로 분석 — 바로 질문하기

설정이 끝나면 채팅창에 **한국어로 그냥 질문**하면 됩니다.

```
질문 예시:
  "2023년 서울에서 부산으로 출근 통행량은?"
  "2023년 전국 수단분담률을 표로 보여줘"
  "2023년 vs 2030년 서울-경기 간 통행량 변화를 비교해줘"
  "2023년 출근 통행 발생량 상위 10개 시군구"
  "부산 → 서울 수단별 통행량을 연도별 추이 표로 만들어줘"
```

**응답 예시**:

| 출발 시도 | 도착 시도 | 연도 | 목적 | 통행량(통행/일) |
|----------|----------|------|------|---------------|
| 서울(1) | 부산(2) | 2023 | 출근 | 12,345 |

> 2023년 기준 서울에서 부산으로의 출근 통행량은 일평균 12,345통행(AAWDT)입니다.
> 이는 주말·공휴일을 제외한 평일 기준 수치입니다.

### 주의사항

| # | 주의 | 대처 |
|---|------|------|
| 1 | **CSV 합계 행 미삭제 시 2배 오류** | 업로드 전 "합계" 행 반드시 삭제 확인 |
| 2 | **대용량 파일 업로드 제한** | 파일이 너무 크면 연도별로 나눠서 업로드 |
| 3 | **AAWDT vs AADT** | AI 응답에 "AAWDT(평일 기준)"임을 명시하도록 지시문에 포함됨 |
| 4 | **수도권 TXT는 직접 변환 어려움** | 수도권 데이터는 개발팀 지원 요청 또는 전국 250존 데이터로 대체 |
