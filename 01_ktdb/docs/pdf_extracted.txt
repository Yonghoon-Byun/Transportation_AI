1. 개요
·
Item 명칭: KTDB Report Agent
·
개발 목적: KTDB 데이터 수집ㆍ가공 자동화를 통한 설계 효율화 및 보고서 품질 
표준화
·
개발 범위: 교통수요예측 기초자료 분석 및 보고서 작성 등에 적용되는
KTDB(사회경제지표, 통행량) 기초 데이터 정리, 데이터 분석, 보고서 내 
표/그래프/분석문구 작성 등
·
AI 적용 영역: 데이터 API 추출(RPA), 자연어 기반 데이터 쿼리(LLM), 데이터 
시각화 및 분석 문구 생성(Generative AI)
·
기대 효과: 기초자료 정리 시간 80% 절감, 데이터 오기입 제로화, 
보고서 작성 속도 2배 향상
2. 현황 및 문제점
·
기존 설계 프로세스 요약
KTDB 데이터 신청 및 다운로드 → 텍스트기반 대용량 O/D 데이터 전처리 → 
고차원 데이터 Dimension Reduction → matrix reformatting(엑셀 전환) →  표 
형식 전산 가공 → (필요시 그래프 작성) → 데이터 분석 및 문구 작성
·
현재 사용 중인 프로그램 및 한계점
파이썬 코딩을 통한 데이터 전처리 과정 : AI 도입으로 초보자도 코딩을 통해 
데이터 전처리 과정을 자동화하고 있으나, 데이터 형태에 따라 오류가 잦고 
다양한 제공 데이터에 적용하는데 한계가 있음
·
인력 소요 및 업무 비효율 분석
표 형태의 데이터 가공을 위해 엑셀 프로그램을 사용함으로써 대용량 데이터 
처리에 따르는 로딩 시간과 데이터 처리에 필요한 인력 및 시간 낭비 발생
업무 형태는 반복적인 작업이나, 과업의 범위나 목적에 따른 변동으로 인해 
비효율적인 반복업무를 지속하고 있음

·
주요 품질/검토 문제 사례
방대한 숫자 데이터를 다루는 과정에서 휴먼 에러(오기입) 발생 가능성이 
상존하며, 분석자의 판단에 의해 데이터 분석 결과를 판단하고 보고서를 
작성하면서 편향된 방향으로 분석할 가능성이 있음
3. 개발 목표 및 추진 전략
·
핵심 목표: 대화형 인터페이스를 통한 교통 데이터 추출/가공 및 보고서용 성과품 
즉시 생성
·
개발 방향:
o
AI 기능 : 자연어로 질문하면 API를 통해 데이터를 호출하는 AI Agent 구축
o
데이터 확보 및 학습 전략 : 부서 내에서 기존에 제공되었던 KTDB 데이터를 
확보 중이며, 향후 비 정기적으로 배포되는 수정 데이터는 다운로드 후 학습 
가능
o
엔지니어–AI 협업 방식(Work Flow Integration) : 엔지니어가 필요한 자료 및 
가공 형태를 질문하면 즉시 결과 도출
예시) “A시 B동의 보고서 현황분석 파트에 들어갈 목적별 통행량과 목적별 
비율 표를 만들고, 분석 문구 작성해줘”
·
성공 지표(KPI):
o
설계 시간 단축 목표 : 데이터 수집 및 가공 시간 단축 : 기존 8시간 → 30분 
이내 단축(90% 개선)
o
자료 생성 만족도 : 실무자 만족도 80% 이상의 긍정 응답
4. 기술 구성 및 구조

엔진]

초안 출력]
