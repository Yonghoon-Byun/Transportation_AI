# 멀티 프로젝트 오케스트레이션 프롬프트 템플릿

## 사용법
1. [대괄호] 안의 내용을 실제 프로젝트에 맞게 수정
2. 오케스트레이션 터미널에서 Step 1 프롬프트 실행
3. 각 서브 프로젝트 터미널에서 Step 2 프롬프트 실행

---

## Step 1: 오케스트레이션 세션 프롬프트 (메인 터미널)

```
[프로젝트 전체 이름] 개발 세션을 구성해줘.

서브 프로젝트 목록:
1) [서브프로젝트1 이름] - [한줄 설명]
2) [서브프로젝트2 이름] - [한줄 설명]
3) [서브프로젝트3 이름] - [한줄 설명]

작업 방식:
- 너는 이 프로젝트들을 묶어서 관리하는 오케스트레이션 에이전트야
- 각 서브 프로젝트는 별도 터미널에서 Claude Code로 병렬 개발할 거야
- 각 서브 프로젝트 폴더에 docs/ 폴더를 만들어줘 (참고 문서를 넣을 곳)
- 각 폴더에 CLAUDE.md, PLAN.md, src/ 기본 구조를 잡아줘

먼저 전체 폴더 구조부터 잡아줘:
- 루트: README.md, .gitignore, shared/, docs/
- 각 서브 프로젝트: CLAUDE.md, PLAN.md, docs/README.md (넣어야 할 문서 가이드), src/__init__.py
```

---

## Step 2: 각 서브 프로젝트 세션 프롬프트 (서브 터미널)

```
이 프로젝트는 [서브프로젝트 이름]이야. [한줄 설명].

docs/ 폴더에 참고 문서를 넣어둘 예정인데, 먼저 docs/ 폴더의 문서가 있으면 모두 읽고, 없으면 PLAN.md를 참고해서 진행해줘.

작업:
1. docs/ 폴더의 참고 문서를 분석
2. 분석 결과를 바탕으로 CLAUDE.md를 구체적으로 재작성
3. PLAN.md를 실제 구현 가능한 상세 계획으로 재작성
4. src/ 하위 모듈 폴더 구조 생성
5. requirements.txt 초안 작성

상위 프로젝트 [루트 프로젝트명]의 오케스트레이션 세션에서 관리 중이고, 이 세션은 [서브프로젝트 이름] 전담이야.
```

---

## Step 3: 참고 문서 추가 후 재분석 프롬프트

```
docs/ 폴더에 참고 문서를 새로 넣었어.
모든 문서를 다시 읽고 CLAUDE.md와 PLAN.md를 문서 내용 기반으로 업데이트해줘.
```

---

## Step 4: 오케스트레이션 세션에서 진행상황 확인 프롬프트

```
각 서브 프로젝트의 현재 상태를 확인해줘.
- 01_[폴더명]/PLAN.md
- 02_[폴더명]/PLAN.md
- 03_[폴더명]/PLAN.md
각각 읽고 진행 상황을 요약해줘.
```

## 실제 적용 예시

### 예시: 환경 모니터링 AI 프로젝트

**오케스트레이션 프롬프트:**
```
환경 모니터링 AI 개발 세션을 구성해줘.

서브 프로젝트 목록:
1) 대기질 예측 모델 - 미세먼지/오존 농도 예측 AI
2) 수질 분석 Agent - 하천 수질 자동 분석 AI Agent
3) 소음 매핑 시스템 - 도시 소음 지도 생성 시스템

작업 방식:
- 너는 이 프로젝트들을 묶어서 관리하는 오케스트레이션 에이전트야

- 각 폴더에 CLAUDE.md, PLAN.md, src/ 기본 구조를 잡아줘

먼저 전체 폴더 구조부터 잡아줘.
```

**서브 세션 프롬프트:**
```
이 프로젝트는 대기질 예측 모델이야. 미세먼지/오존 농도를 예측하는 AI.

docs/ 폴더에 참고 문서를 넣어둘 예정인데, 먼저 docs/ 폴더의 문서가 있으면 모두 읽고, 없으면 PLAN.md를 참고해서 진행해줘.

5. requirements.txt 초안 작성

상위 프로젝트 Environmental_AI의 오케스트레이션 세션에서 관리 중이고, 이 세션은 대기질 예측 전담이야.
```
