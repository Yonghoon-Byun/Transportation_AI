# 오케스트레이션 가이드

## 병렬 개발 워크플로우

각 서브 프로젝트는 VSCode에서 독립된 Claude Code 세션으로 개발합니다.

### 세션 구성

| 세션 | 작업 폴더 | 담당 |
|------|----------|------|
| 메인 (현재) | `Transportation_AI/` | 오케스트레이션, 공통 구조 |
| 세션 1 | `Transportation_AI/01_ktdb/` | KTDB 데이터 분석 |
| 세션 2 | `Transportation_AI/02_geotechnical_agent/` | 지반조사 분석 Agent |
| 세션 3 | `Transportation_AI/03_railway_optimization/` | 철도 선형 최적화 |

### 작업 순서

1. **구조 수립** (메인 세션) - 완료 후 각 세션 시작
2. **병렬 계획** (세션 1~3) - 각 프로젝트별 상세 계획
3. **병렬 개발** (세션 1~3) - 독립적으로 개발 진행
4. **통합 검증** (메인 세션) - 공통 인터페이스 확인

### 세션 간 공유 사항
- `shared/` 폴더: 공통 유틸리티 (로깅, 설정 관리 등)
- `docs/`: 전체 프로젝트 문서
- 각 프로젝트의 CLAUDE.md: 세션별 컨텍스트
