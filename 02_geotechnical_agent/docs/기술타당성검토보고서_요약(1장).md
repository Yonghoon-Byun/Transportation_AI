# 지반조사 자동화 도구 개발 - 기술 타당성 검토 요약

**작성일**: 2026년 3월 20일
**목적**: 개발 착수 의사결정 지원 (설계엔지니어링 실무자용 요약본)
**원본**: 기술타당성검토보고서.md (상세 기술 검토 결과)

---

## 무엇을 만드는가

지반조사 보고서(PDF)를 입력하면, 시추공 데이터를 자동으로 읽어 지반정수를 산정하고 엑셀 요약표와 그래프를 출력하는 자동화 도구를 개발한다. 엔지니어가 보고서를 직접 읽고 손으로 정리하던 반복 작업을 디지털 조수가 대신 처리하되, 최종 확인은 엔지니어가 수행하는 구조다.

---

## 만들 수 있는가

**[노란불] 조건부 개발 가능 - 단계적 접근 필요**

기술적으로 구현은 가능하나, 보고서(PDF)에서 데이터를 자동으로 읽는 부분이 예상보다 어렵다. 1차 완성본에서 즉시 목표 수준에 도달하기는 어려우며, 사용하면서 점진적으로 정확도를 높여가는 방식으로 진행해야 한다. 비유하자면, 처음부터 완벽한 자동화가 아니라 "80점짜리 자동화 + 엔지니어 검토"로 시작하는 것이다.

---

## 주요 검토 결과

| 구분 | 내용 | 비고 |
|------|------|------|
| 개발 기간 | 약 4개월 | 기획서 원안(약 2.5개월) 대비 1.5개월 추가 필요 |
| 자동화 수준 (1차) | 업무시간 약 50~60% 단축 | 원안 목표 70%는 6개월 이후 달성 가능 |
| 운영 비용 | 월 수십만원 수준 | 과업 건수에 따라 변동 (월 10만~150만원 범위) |
| 관공서 납품 | 별도 수작업 변환 필요 | 한글(HWP) 파일 자동 생성 불가, Word로 출력 후 변환 |

---

## 주의할 점

- **가장 어려운 부분은 보고서 읽기다.** 표 구조가 정리된 디지털 보고서는 잘 처리되지만, 그래픽 레이아웃의 주상도나 오래된 스캔본 문서는 자동 인식 오류가 잦다. 도구가 못 읽는 경우를 위한 수동 입력 경로를 함께 마련해야 한다.
- **자동 산출값은 반드시 엔지니어가 검토해야 한다.** 프로그램이 경험식을 잘못 적용하거나 잘못된 수치를 내놓을 수 있다. 도구의 결과를 최종 설계값으로 그대로 사용해서는 안 된다.
- **관공서 과업에는 사전 확인이 필요하다.** 지반조사 데이터를 해외 AI 서비스로 전송하는 구조이므로, 발주처의 데이터 보안 정책과 저촉되지 않는지 과업 착수 전에 확인해야 한다.

---

## 착수 전 필요한 것

- [ ] 실제 지반조사 PDF 10건 이상 확보 (3개 이상 업체 양식 포함)
- [ ] 확보한 PDF에 대해 엔지니어가 수작업으로 정답 데이터 작성 (약 2주 소요)
- [ ] 지반공학 실무 경력 5년 이상 전문가 1명 개발 초기부터 참여 확보
- [ ] AI 서비스 사용 비용 예산 반영 (월 수십만원, 개발 기간 포함 약 4개월분)
