### 3.2 심도-N치 프로파일 해석

```
심도 (m)  N치
0         —
2         6   ← 매립층 하단
3         11  ← 풍화토 상부 (느슨~보통)
5         34  ← 풍화토 중부 (조밀)
7         45  ← 풍화토 하부 (매우 조밀)
9         50* ← 반발 시작 (풍화도 낮은 풍화토)
11        50*
13        50*
13+       50* ← 풍화암 전 구간 반발
```
*50은 관입불능(반발)

- **지반 강도 전이 심도**: 약 GL 7~9 m 구간에서 N치 급증 (45→50반발)
- **실무 적용**: 케이싱 2.5 m로 매립층만 보호 → 풍화토 상부 굴착 시 공벽 붕괴 주의 (N=11, 느슨)
- **SPT 분석 한계**: 풍화암 이하는 N치 대신 TCR/RQD로 암반 품질을 평가해야 함

## 6. AI Agent 파서 개발 시사점

### 6.1 PDF 구조 특성 및 OCR 난이도

NTB-24 주상도 PDF는 총 4페이지로 구성되며, 각 페이지의 성격이 상이하다.

| 페이지 | 주요 내용 | OCR 난이도 | 주요 어려움 |
|---|---|---|---|
| Page 1 | 기본정보 헤더 + 토사층 주상도 | 중(Medium) | 헤더 테이블 비정형, N치 그래픽 위치 기반 |
| Page 2 | 풍화암-경암 전이, 터널 시점 표시 | 높음(High) | 빨간색 마킹 텍스트 인식, 코어 사진 영역 혼재 |
| Page 3 | 경암 중간부, 터널 종점 표시 | 높음(High) | 연속된 암반 기술 칼럼, 빨간색 마킹 |
| Page 4 | 경암 하부, 시추 종료 | 중(Medium) | 심도 연속성, 표 경계 불명확 |

**핵심 OCR 과제:**
1. **N치 위치 기반 추출**: N치 값이 텍스트가 아닌 그래픽(막대 위치)으로 표현될 수 있음. 실제 숫자(예: "6/30")는 옆에 텍스트로 존재하나 심도 연동이 필요.
2. **반발 표기 다양성**: "50/18(반발)", "50/7 R", "REFUSAL" 등 표기 방식이 문서마다 다를 수 있음. 정규화 필수.
3. **심도 연속성 체크**: 4페이지에 걸쳐 심도가 연속(0→70 m)되므로 페이지 경계에서 심도 이어받기 로직 필요.
4. **컬러 마킹 인식**: 터널 시점/종점은 빨간색 텍스트로 표시됨 → `Claude Vision` 기반 multimodal 접근이 유리.
5. **파쇄대 특기사항 파싱**: "28.1~29.8m 파쇄 및 core loss" 형태의 인라인 텍스트를 구조화된 필드로 추출하는 NLP 처리 필요.
6. **TCR/RQD 누락 처리**: 일부 구간에서 명시적 수치 없이 코어 사진만 있을 수 있음. `None` 처리 기준 명확화 필요.

### 6.2 필드 추출 우선순위 및 방법

| 추출 필드 | 위치 패턴 | 추출 방법 | 신뢰도 |
|---|---|---|---|
| 시추공 번호 | 상단 헤더 고정 위치 | 정규식 `[A-Z]+-\d+` | 높음 |
| 시추표고 | 헤더 테이블 | 키-값 쌍 | 높음 |
| 시추심도 | 헤더 테이블 | 키-값 쌍 | 높음 |
| 지하수위 | 헤더 테이블 | 키-값 쌍 | 높음 |
| 지층 경계 심도 | 주상도 좌측 눈금 | 위치 기반 + OCR | 중간 |
| N치 (수치) | 주상도 그래픽 옆 텍스트 | OCR + 정규식 | 중간 |
| N치 반발 | "50/N(반발)" 패턴 | 정규식 + 의미 처리 | 중간 |
| TCR/RQD | 암반 코어 구간 테이블 | 표 파싱 (tabula-py) | 중간 |
| 파쇄대 위치 | 기술란 텍스트 | NLP + 정규식 | 낮음 |
| 터널 시·종점 | 컬러 마킹 텍스트 | Claude Vision | 낮음 |

### 6.3 파서 구현 권장 전략

```
1단계: pdfplumber → 텍스트 레이어 추출 (헤더 정보, 테이블 구조)
2단계: tabula-py  → 표 형식 데이터 추출 (SPT 기록, TCR/RQD 테이블)
3단계: Claude Vision (API) → 그래픽 요소 해석
       - 주상도 이미지 → 지층 경계, N치 위치, 파쇄대 기호
       - 컬러 마킹 텍스트 → 터널 시·종점
4단계: NLP 후처리 → 비고란 자유텍스트 구조화
       - 파쇄대 심도 추출: r'(\d+\.?\d*)~(\d+\.?\d*)m\s*(파쇄|파쇄대)'
       - 절리 방향 추출: r'[∠∠](\d+)'
5단계: 데이터 검증 → AnomalyWarning 생성
       - 심도 단조증가, 층 두께 합산, N치 범위 등
```

### 7.2 SoilLayer 매핑

| schemas.py 필드 | PDF 원본 | 매립층 | 풍화토 | 풍화암 | 경암 |
|---|---|---|---|---|---|
| `layer_no: int` | (순서) | `1` | `2` | `3` | `4` |
| `soil_name: str` | 지층명 | `"매립층"` | `"풍화토"` | `"풍화암"` | `"경암"` |
| `depth_from: float` | 지층 상단 | `0.0` | `2.0` | `13.0` | `22.0` |
| `depth_to: float` | 지층 하단 | `2.0` | `13.0` | `22.0` | `70.0` |
| `thickness: float` | 두께 | `2.0` | `11.0` | `9.0` | `48.0` |
| `spt_n_values: list[int]` | N치 목록 | `[6]` | `[11,34,45]` | `[]` | `[]` |
| `n_avg: float` | 평균 N치 | `6.0` | `30.0` | `None` | `None` |
| `is_rock: bool` | 암반 여부 | `False` | `False` | `True` | `True` |
| `rqd: float` | RQD 평균 | `None` | `None` | (측정됨) | `88.3` |
| `tcr: float` | TCR 평균 | `None` | `None` | (측정됨) | `99.5` |
| `color: str` | 색상 | `"암갈색"` | `"황갈색"` | `None` | `"암회색~갈회색"` |
| `description: str` | 상태 기술 | `"자갈섞인 실트질 모래"` | `"실트질 모래, 느슨~매우 조밀"` | `"화강암 풍화, 실트질 모래로 분해"` | `"화강편마암, 신선, 강한~매우 강한"` |

**설계 상 이슈:**
- `rqd`, `tcr`은 `SoilLayer` 단위로 평균값을 저장하지만, 실제 데이터는 3 m 간격 측정값의 시계열이므로 `spt_records`처럼 별도 레코드 리스트(`rock_core_records`)가 필요할 수 있음.
- `is_rock=True`인 풍화암에서 N치 측정값이 없으므로 `spt_n_values=[]`로 처리. CLAUDE.md 규칙대로 `None` 아닌 빈 리스트 사용 올바름.

### 7.3 SPTRecord 매핑

| schemas.py 필드 | PDF 원본 | 예시 값 | 파싱 규칙 |
|---|---|---|---|
| `depth: float` | 측정 심도 | `9.0` | 주상도 눈금 위치 기반 |
| `n_value: int` | N치 (타격수) | `50` | "50/18" → 분자 50이 N치 |
| `remarks: str` | 비고 | `"반발(18cm)"` | "50/18" 분모 → 실제 관입량 (cm) |

**반발 처리 규칙 (스키마 명시 필요):**
- `n_value=50` + `remarks="반발(관입량xcm)"` 형태로 표준화
- `schemas.py`의 `remarks` docstring에 "관입불능" 예시가 이미 있어 일관성 있음
- 단, 실제 데이터에서 "50/7", "50/13", "50/15", "50/18"는 모두 다른 관입 저항을 나타내므로 분모값을 `remarks`에 보존해야 함

### 7.4 스키마 확장 권고사항

현재 `schemas.py`에서 NTB-24 데이터를 완전히 수용하려면 다음 보완이 필요하다:

**단기 (파싱 정확도 직결):**

```python
@dataclass
class RockCoreRecord:
    """암반 코어 기록 (3m 런 단위)."""
    depth_from: float          # 런 상단 심도 (m)
    depth_to: float            # 런 하단 심도 (m)
    tcr: float                 # Total Core Recovery (%)
    rqd: float                 # Rock Quality Designation (%)
    joint_sets: Optional[int]  # 절리 세트 수
    joint_angles: list[int]    # 절리 경사각 목록 (°)
    fracture_zones: list[tuple[float, float]]  # 파쇄대 구간 [(from, to), ...]
    remarks: Optional[str]     # 방해석 충진, 산란 등
```

**중기 (프로젝트 관리 연동):**

```python
@dataclass
class BoreholeLog:  # 확장
    # ... 기존 필드 ...
    project_name: Optional[str] = None      # 건 명
    structure_name: Optional[str] = None    # 구조물명
    drill_date: Optional[str] = None        # 조사일 (ISO 8601)
    drilling_method: Optional[str] = None   # 시추방법
    casing_depth: Optional[float] = None    # 케이싱 심도 (m)
    drill_diameter: Optional[str] = None    # 시추공경 (NX, HX 등)
    rock_core_records: list[RockCoreRecord] = field(default_factory=list)
    tunnel_start_depth: Optional[float] = None  # 터널 시점 심도
    tunnel_end_depth: Optional[float] = None    # 터널 종점 심도
```

## 8. 결론 및 시사점

### 8.1 지반공학적 결론

1. **지반 구성**: 매립층(2 m) → 풍화토(11 m) → 풍화암(9 m) → 경암(48 m)의 전형적인 한반도 화강암 지대 층서 구조.
2. **터널 암반 품질**: 터널 구간(GL -21.87 ~ -55 m) 전반에 걸쳐 RQD 82~95%, TCR 95~100%의 양호한 암반이나, GL -28~30 m 구간 파쇄대(170 cm)가 설계상 핵심 취약점.
3. **지하수**: GL -13.2 m의 지하수위는 터널 굴착 전 구간에 영향. 방해석 충진 절리(58~64 m)는 과거 열수 활동 흔적으로 국부 용수 가능성을 시사.
4. **설계 정수 적용**: 경암 구간은 RMR II등급 추정, 터널 지보 패턴 결정에 충분한 암반 품질. 단, 파쇄대 구간은 별도 보강 검토 필요.

### 8.2 AI Agent 파서 개발 우선과제

1. **즉시**: `RockCoreRecord` 모델 추가 및 `BoreholeLog` 확장 필드 구현
2. **단기**: 반발 N치 정규화 파이프라인 (`"50/18"` → `n_value=50, remarks="반발(18cm)"`)
